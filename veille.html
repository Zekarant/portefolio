Veille technologique : Faille XSS


Voici la veille technologique que j'ai décidé de traiter pour mon épreuve de BTS SIO.

Que-ce que la faille XSS ?

XSS est l'abréviation de "Cross-Site Scripting".

Faille avec comme principe d'injecter du code directement dans le navigateur web dans le but de récupérer des données sensibles et personnelles. Elle agit du côté du client et non du serveur.
Cette faille est présente dans les sites Internet ne proposant pas de sécurité sur les données envoyées par les utilisateurs. Bien que très facile à contrer, elle est l'une des failles les plus importantes du web.

Comment provoquer une faille XSS ?

Il s'agit d'une modification de l'URL ou de l'envoi de formulaire permettant de récupérer les données et y ajouter des données/commandes/morceaux de code dans le but de récupérer les données des utilisateurs.

Il est possible d'utiliser du code JavaScript ou de créer des cookies pour dérober les données sensibles des utilisateurs.

Plusieurs types de faille XSS :

  • La faille XSS non permanente (Les données ne sont pas stockées, et nous ne nous servons des données que de l'utilisateur sur le moment)
  • La faille XSS permanente (Les données sont stockées et tous les utilisateurs peuvent subir la faille en se rendant sur un forum par exemple)

Les grandes actualités de la faille XSS :

2016 - Yahoo : Yahoo laisse une faille XSS laissant à disposition les mails de tous les utilisateurs.

2017 - Cisco : Faille permettant l'exécution du code à distance.

2017 - Steam : Faille permettant d'injecter du code dans les profils de tous les utilisateurs de la plateforme de jeux vidéo.

2018 - IBM : Failles qui permettaient à un utilisateur d'utiliser les privilèges root à distance sur des produits vulnérables. Les failles permettaient de contourner l'authentification, l'autre permettaient d'exécuter et télécharger un shell. La troisième pouvait donner les droits root.

2020 - Discord : Faille permettant l'exécution du code à distance à cause de paramètre NodeJS mal paramétrés.
